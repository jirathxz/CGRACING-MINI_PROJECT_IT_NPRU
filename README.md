# CG RACING - E-Commerce Web Concept

โครงการพัฒนาเว็บไซต์ E-Commerce ต้นแบบสำหรับการจัดจำหน่ายผลิตภัณฑ์ดูแลรักษารถยนต์และน้ำมันเครื่อง โครงการนี้เน้นการออกแบบส่วนติดต่อผู้ใช้ (UI) ด้วยธีมสีเข้ม (Dark Mode) และการจัดการประสบการณ์ผู้ใช้ (UX) ให้มีความลื่นไหล ทันสมัย และตอบสนองต่อทุกขนาดหน้าจอ

---

## 🛠 เทคโนโลยีและสถาปัตยกรรมที่ใช้

* **HTML5:** โครงสร้างหลักของเว็บไซต์ มีการแบ่งแยกส่วนประกอบ (Component-based approach) อย่างชัดเจน
* **CSS3:** * ควบคุมธีมสีหลักผ่าน **CSS Variables** (เช่น `--bg-main`, `--accent`)
  * จัดวางเลย์เอาต์ด้วย **Flexbox** และ **CSS Grid**
  * ผสานการทำงานกับแบบอักษร 'Prompt' สำหรับข้อความทั่วไป และ 'Novusa' สำหรับโลโก้แบรนด์
* **Vanilla JavaScript:** * จัดการการแสดงผลข้อมูลแบบไดนามิก (Dynamic Rendering)
  * จัดการระบบ **Fetch API** สำหรับโหลดคอมโพเนนต์ส่วนนำทาง
  * ควบคุมระบบโต้ตอบของผู้ใช้ เช่น การสไลด์ภาพ และการรองรับการสัมผัส (Touch Events)
* **Font Awesome:** ใช้สำหรับการแสดงผลไอคอนสัญลักษณ์ทั่วทั้งเว็บไซต์

---

## ✨ คุณสมบัติเด่นของระบบ

### 1. Dynamic Component Loading
ระบบใช้ JavaScript Fetch API ในการโหลดส่วน Header และ Sidebar จากไฟล์ `nav.html` เข้ามาแสดงผลในหน้าเว็บต่าง ๆ ช่วยให้การแก้ไขเมนูทำได้จากจุดเดียวและลดความซ้ำซ้อนของโค้ด

### 2. Intelligent Sidebar Navigation
แถบนำทางด้านข้างออกแบบให้ประหยัดพื้นที่ในสภาวะปกติ (Collapsed) และจะขยายขนาดเป็น 180px พร้อมแสดงชื่อเมนูเมื่อมีการวางเมาส์ (Hover Effect) นอกจากนี้ยังมีการตรวจสอบ URL ปัจจุบันเพื่อเน้นสีเมนูที่กำลังใช้งานอยู่ (Active State)

### 3. Advanced Hero Slider
ระบบแบนเนอร์โฆษณาในหน้าหลัก (`index.html`) ทำงานอย่างสมบูรณ์แบบด้วยฟังก์ชัน:
* **Auto-Play:** เปลี่ยนภาพอัตโนมัติทุก 5 วินาที
* **Touch Support:** รองรับการปัดซ้าย/ขวา (Swipe) บนอุปกรณ์พกพาผ่านการคำนวณ `startX` และ `endX`
* **Manual Control:** ควบคุมการเปลี่ยนภาพผ่านจุดนำทาง (Navigation Dots)

### 4. Client-side Product Rendering
ข้อมูลสินค้าถูกแยกการจัดเก็บไว้ใน `products.js` ในรูปแบบ Array of Objects จำนวน 30 รายการ และใช้ JavaScript ประมวลผลเพื่อสร้างโครงสร้าง HTML (Product Cards) เข้าไปในหน้าเว็บแบบไดนามิก รองรับการแสดงผลทั้งราคา หมวดหมู่ คะแนนรีวิว และจำนวนยอดขาย

### 5. Responsive Design
โครงสร้างเว็บออกแบบมาให้รองรับหน้าจอทุกขนาด (Breakpoints ที่ 768px) 
* **Desktop:** แสดงผลครบถ้วนเต็มรูปแบบ
* **Mobile/Tablet:** ระบบจะทำการซ่อน Sidebar และขยายพื้นที่เนื้อหาให้เหมาะสมกับหน้าจอขนาดเล็กอัตโนมัติ

## 📂 โครงสร้างไฟล์ในโครงการ

* `index.html`: หน้าหลัก แสดงแบนเนอร์สไลด์ หมวดหมู่ยอดนิยม และดึงรายการสินค้ามาแสดงผล
* `discover.html`: หน้าค้นหาและเลือกดูสินค้า พร้อมระบบปุ่มกรองหมวดหมู่ (UI)
* `nav.html`: ไฟล์คอมโพเนนต์ส่วนกลาง ประกอบด้วย Header (ช่องค้นหา, เมนูผู้ใช้) และ Sidebar
* `style.css`: ไฟล์สไตล์ชีตหลัก จัดการตัวแปรสี ฟอนต์ และเลย์เอาต์ของทุกหน้า
* `products.js`: ไฟล์จำลองฐานข้อมูล (Mock Database) จัดเก็บข้อมูลสินค้า 30 รายการ
* `fonts/`: โฟลเดอร์สำหรับจัดเก็บฟอนต์พิเศษ (Novusa.otf)

## 🚀 วิธีการติดตั้งและใช้งาน

**ข้อควรระวัง:** เนื่องจากโปรเจกต์นี้มีการใช้งาน `fetch()` API สำหรับดึงไฟล์ `nav.html` การเปิดไฟล์ HTML โดยตรงผ่านบราวเซอร์ (file://) อาจทำให้เกิดปัญหา CORS (Cross-Origin Resource Sharing) และทำให้เมนูไม่แสดงผล

**วิธีที่แนะนำ:**
1. ติดตั้งและใช้งานผ่าน Local Server
   * หากใช้ **VS Code**: แนะนำให้ติดตั้ง Extension `Live Server` และคลิกขวาที่ `index.html` เลือก "Open with Live Server"
   * หรือใช้ Python: รันคำสั่ง `python -m http.server` ในโฟลเดอร์โปรเจกต์
2. เข้าถึงเว็บไซต์ผ่าน `http://localhost:XXXX` ตามที่ Local Server กำหนด
