# 🏁 CGRACING - E-Commerce Web Application
**มินิโปรเจกต์เว็บไซต์ร้านค้าออนไลน์สำหรับคนรักรถ (HTML, CSS, JS)**

CGRACING คือแพลตฟอร์ม E-Commerce จำลองสำหรับจำหน่ายน้ำมันเครื่องและอุปกรณ์ดูแลรักษารถ โปรเจกต์นี้ถูกพัฒนาขึ้นมาโดยเน้นความสมบูรณ์แบบทั้ง 2 ด้าน คือ **ระบบการทำงานฝั่ง Front-end ที่สมจริง** และ **การออกแบบ UX/UI ที่สวยงามทันสมัย** เพื่อสร้างประสบการณ์การใช้งานที่ดีที่สุด

---

## 🚀 ระบบการทำงานหลัก และไฟล์ที่เกี่ยวข้อง (Core Features)

เราได้แบ่งการทำงานของระบบออกเป็น 4 ส่วนหลัก โดยใช้ JavaScript ในการจัดการ Logic และ LocalStorage ในการเก็บข้อมูล ดังนี้ครับ:

### 1. ระบบรักษาความปลอดภัยและการยืนยันตัวตน (Authentication)
* **ล็อกอินเพื่อแยกสิทธิ์:** ผู้ใช้ต้องผ่านหน้า `login.html` โดยระบบจะเช็ค Username/Password (ตั้งค่าเริ่มต้นไว้ที่ `cgracing` / `6744`)
* **ป้องกันการแอบเข้าถึง (Route Protection):** เราใช้สคริปต์ `auth.js` เป็นเสมือนยามเฝ้าประตู หากใครพยายามเข้าหน้าเว็บโดยไม่ได้ล็อกอิน สคริปต์ตัวนี้จะเตะกลับไปหน้า Login ทันที พร้อมใช้ `localStorage` จดจำสถานะผู้ใช้ไว้

### 2. ระบบค้นหาและแสดงผลสินค้า (Storefront & Search)
* **เรนเดอร์ข้อมูลแบบไดนามิก:** สินค้าทั้งหมดไม่ได้ถูกเขียนติดไว้ในหน้าเว็บ (Hardcode) แต่เราสร้างฐานข้อมูลจำลองเป็นไฟล์ `products.js` แล้วใช้ JavaScript ดึงมาสร้างเป็นการ์ดสินค้าบนหน้า `index.html` และ `discover.html`
* **ระบบกรองและค้นหา (Real-time Filter):** เมื่อผู้ใช้กดเลือกหมวดหมู่ หรือพิมพ์ค้นหาในหน้า `discover.html` สินค้าจะถูกคัดกรองและแสดงผลทันทีโดยไม่ต้องโหลดหน้าเว็บใหม่
* **หน้ารายละเอียดสินค้า:** เมื่อคลิกที่สินค้า ระบบจะส่งข้อมูลไปที่หน้า `view_product.html` เพื่อแสดงรูปภาพ ราคา และรายละเอียดแบบเจาะจง

### 3. ระบบตะกร้าสินค้าและการสั่งซื้อ (Shopping Cart)
* **เพิ่มสินค้าลงตะกร้า:** เมื่อผู้ใช้กดเพิ่มสินค้าจากหน้ารายละเอียด โค้ดจะจัดการแปลงข้อมูลเป็น JSON แล้วเก็บลงใน `localStorage`
* **แจ้งเตือนแบบเรียลไทม์:** ตัวเลขบนไอคอนตะกร้าในแถบเมนู (`nav.html`) จะอัปเดตอัตโนมัติทุกครั้งที่มีการหยิบของใส่ตะกร้า
* **คำนวณเงินและเช็คเอาต์:** หน้า `product_cart.html` จะดึงข้อมูลมาแสดงผล ผู้ใช้สามารถกดเพิ่ม/ลดจำนวนได้ (ยอดเงินรวมจะคำนวณใหม่ทันที) และเมื่อกดชำระเงิน จะมีหน้าต่าง Pop-up ยืนยัน พร้อมเคลียร์ข้อมูลตะกร้าให้ว่าง

### 4. ระบบหลังบ้านสำหรับผู้ขาย (Seller Dashboard)
* **หน้าจัดการสต็อก:** หน้า `seller.html` ถูกออกแบบมาให้ผู้ขายดูภาพรวมสินค้าได้ง่ายผ่านตาราง Data Table
* **ระบบจำลองการจัดการ (CRUD):** มีปุ่มฟังก์ชันสำหรับ เพิ่ม (Create), แก้ไข (Update), และ ลบ (Delete) สินค้า โดยเฉพาะการลบ จะมีการดึงชื่อสินค้ามาแสดงในหน้าต่างยืนยันเพื่อป้องกันความผิดพลาด

## 🎨 การออกแบบ UX/UI (User Experience & User Interface Design)

ในส่วนของการนำเสนอภาพลักษณ์ เราใช้หลักการออกแบบการสื่อสารด้วยภาพ (Visual Communication Design) เพื่อคุมโทนเว็บไซต์ให้ดูแพงและใช้งานง่าย:

* **การคุมโทนสี (Color Palette):** เราเลือกใช้ Dark Theme พื้นหลังสีเข้ม (`#15141b`) ตัดกับสีแดง Accent Color (`#e24242`) ซึ่งเป็นสีแห่งความเร็วและสปอร์ต ช่วยนำสายตาไปยังปุ่ม Call-to-Action สำคัญๆ ได้ดีเยี่ยม
* **การจัดวางตัวอักษร (Typography):** ใช้ฟอนต์ไม่มีหัวอย่าง **Prompt** ทั้งระบบเพื่อความเรียบหรู อ่านง่าย และใช้ฟอนต์ **Novusa** สำหรับโลโก้ `CG RACING` เพื่อสร้าง Brand Identity ที่แข็งแกร่ง
* **ประสบการณ์การใช้งานบนมือถือ (Mobile-First):** ไฟล์ `style.css` ถูกเขียนให้รองรับทุกหน้าจอ (Responsive) จุดเด่นคือเมื่อเปิดในมือถือ เมนูนำทางด้านซ้ายจะถูกซ่อนไว้ แล้วเปลี่ยนเป็นแถบเมนูด้านล่าง (Bottom Navigation) แทน ให้ความรู้สึกเหมือนกำลังใช้แอปพลิเคชัน
* **แอนิเมชันและเอฟเฟกต์ (Micro-interactions):** การใส่เทคนิค Glassmorphism (พื้นหลังเบลอโปร่งแสง) และแอนิเมชันรูปสินค้าขยับขึ้นลงเบาๆ บนแบนเนอร์ ช่วยเพิ่มชีวิตชีวาให้เว็บไซต์ไม่ดูแข็งทื่อจนเกินไป
* **หน้าต่างแจ้งเตือนสุดสมาร์ท:** เราเลิกใช้หน้าต่าง Alert แบบเก่า แล้วฝังไลบรารี `SweetAlert2` เข้าไปแทน ทำให้ข้อความแจ้งเตือนต่างๆ ดูสวยงามและกลมกลืนกับธีมของเว็บ

## 🛠️ โครงสร้างเทคโนโลยี (Tech Stack)
โปรเจกต์นี้เขียนขึ้นด้วยเทคโนโลยีพื้นฐานของ Web Development รันผ่าน Client-side 100%
* **หน้าบ้าน (Frontend):** HTML5, CSS3 (ใช้งาน CSS Variables) และ Vanilla JavaScript (ES6+)
* **ฐานข้อมูล (Mock Database):** LocalStorage & JSON (ในไฟล์ `products.js`)
* **ส่วนเสริม (Libraries):** FontAwesome (ไอคอน), SweetAlert2 (หน้าต่างแจ้งเตือน)

## 📂 โครงสร้างไฟล์ทั้งหมด (File Structure)
เพื่อให้ง่ายต่อการแก้ไขและดูแลรักษา เราได้แยกส่วนประกอบต่างๆ ออกเป็นไฟล์ดังนี้:

* `index.html` / `discover.html` / `view_product.html` - กลุ่มหน้าแสดงผลสินค้า
* `product_cart.html` / `seller.html` - กลุ่มหน้าต่างจัดการตะกร้าและระบบหลังบ้าน
* `login.html` / `register.html` / `contact.html` - กลุ่มหน้ายืนยันตัวตนและติดต่อ
* `nav.html` - ไฟล์แถบเมนู (ใช้ JavaScript Fetch ไปแสดงผลทุกหน้า ประหยัดเวลาแก้โค้ดซ้ำ)
* `style.css` - ไฟล์ควบคุมความสวยงามและเลย์เอาต์ทั้งหมด
* `auth.js` / `products.js` - ไฟล์ควบคุม Logic การล็อกอิน และเก็บคลังข้อมูลสินค้า
