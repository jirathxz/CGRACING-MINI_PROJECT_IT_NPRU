# CGRACING - E-Commerce Web Application (Mini-Project)

แพลตฟอร์มร้านค้าออนไลน์สำหรับจำหน่ายน้ำมันเครื่อง อะไหล่ และอุปกรณ์ดูแลรักษารถยนต์/มอเตอร์ไซค์ ที่ถูกออกแบบมาโดยเน้นประสบการณ์ผู้ใช้ (UX) และส่วนติดต่อผู้ใช้ (UI) ที่ทันสมัย รองรับการแสดงผลทุกหน้าจอ (Responsive Design) ขับเคลื่อนการทำงานฝั่งไคลเอนต์ด้วย HTML, CSS, และ Vanilla JavaScript

## 🚀 ฟีเจอร์และระบบการทำงานหลัก (Core Features & Systems)

โปรเจกต์นี้ประกอบด้วยระบบหลัก 4 ส่วน ดังนี้:

### 1. ระบบรักษาความปลอดภัยและการยืนยันตัวตน (Authentication System)
ระบบควบคุมการเข้าถึงหน้าเว็บต่างๆ เพื่อแยกสิทธิ์การใช้งานระหว่างผู้เยี่ยมชมและผู้ใช้งานที่ล็อกอินแล้ว
* **Login Flow:** หน้า `login.html` ตรวจสอบชื่อผู้ใช้และรหัสผ่าน (Hardcoded สำหรับโปรเจกต์: User= `cgracing`, Pass= `6744`)
* **Session Management:** ใช้ `localStorage` (`isLoggedIn`, `userEmail`) ในการจดจำสถานะการล็อกอิน
* **Route Protection:** ไฟล์ `auth.js` ทำหน้าที่เป็น Middleware ตรวจสอบสถานะ หากยังไม่ได้ล็อกอินและพยายามเข้าถึงหน้าจัดการ จะถูก Redirect กลับไปที่หน้า Login ทันที

### 2. ระบบค้นหาและแสดงผลสินค้า (Product Display & Discovery System)
ระบบดึงข้อมูลจำลองจากไฟล์ `products.js` (Mock Database) มาเรนเดอร์บนหน้าเว็บแบบไดนามิก
* **Dynamic Rendering:** หน้า `index.html` และ `discover.html` สร้างการ์ดสินค้าอัตโนมัติตามจำนวนข้อมูลที่มี
* **Category Filtering:** ปุ่มกรองหมวดหมู่สินค้า (เช่น น้ำมันเครื่อง, ดูแลรักษารถ) ที่ทำงานแบบเรียลไทม์ (Real-time Filtering)
* **Search Engine:** แถบค้นหาสินค้าที่สามารถจับ Keyword และอัปเดต URL Parameters ทำให้สามารถแชร์ลิงก์ผลการค้นหาได้
* **Product Details:** หน้า `view_product.html` รับข้อมูลสินค้าที่ผู้ใช้เลือกผ่าน `localStorage` เพื่อแสดงรายละเอียด ราคา รูปภาพ และเรตติ้ง

### 3. ระบบตะกร้าสินค้าและการสั่งซื้อ (Shopping Cart System)
ระบบจัดการคำสั่งซื้อฝั่งผู้ใช้งานที่รวดเร็วและตอบสนองทันที
* **Add to Cart:** เพิ่มสินค้าจากหน้ารายละเอียด ระบุจำนวน และเก็บข้อมูลโครงสร้าง JSON ลงใน `localStorage` (`productCart`)
* **Cart Management:** หน้า `product_cart.html` ดึงข้อมูลมาแสดงผล ผู้ใช้สามารถกดเพิ่ม/ลดจำนวนสินค้า (Quantity) หรือลบสินค้าออกจากตะกร้าได้
* **Real-time Calculation:** ระบบคำนวณยอดรวมสินค้า (Sub Total) และยอดสุทธิ (Grand Total) อัตโนมัติทุกครั้งที่มีการเปลี่ยนแปลง
* **Notification Badge:** ตัวเลขแจ้งเตือนบนไอคอนตะกร้าใน Navbar (อัปเดตแบบเรียลไทม์ด้วย `setInterval`)
* **Checkout Simulation:** แจ้งเตือนยืนยันการสั่งซื้อและเคลียร์ตะกร้าเมื่อชำระเงินสำเร็จ (ผ่าน SweetAlert2)

### 4. ระบบหลังบ้านสำหรับผู้ขาย (Seller Management System)
หน้า Dashboard (`seller.html`) สำหรับจัดการคลังสินค้า
* **Data Table:** แสดงรายการสินค้าทั้งหมดในรูปแบบตารางที่ดูง่าย พร้อมภาพประกอบและหมวดหมู่
* **Inventory Filtering:** ระบบกรองสินค้าตามหมวดหมู่และช่องค้นหาสำหรับผู้ขาย
* **CRUD Operations (Mockup):** * **C**reate: ปุ่มเพิ่มสินค้าใหม่
    * **U**pdate: ปุ่มแก้ไขข้อมูลสินค้า (ดึงชื่อสินค้ามาประมวลผล)
    * **D**elete: ปุ่มลบสินค้าพร้อมหน้าต่างยืนยันเพื่อป้องกันการกดผิด

## 🎨 การออกแบบ UX/UI (User Experience & User Interface Design)

ระบบถูกออกแบบด้วยความใส่ใจในสุนทรียภาพ (Aesthetics) และการใช้งานที่ลื่นไหล (Usability) โดยมีจุดเด่นดังนี้:

### 1. โทนสีและสไตล์ (Color Palette & Dark Theme)
* **Modern Dark Theme:** ใช้พื้นหลังโทนสีเข้ม (`#15141b` และ `#201e27`) เพื่อความสบายตา ลดความเหนื่อยล้าจากการมองจอ และทำให้รูปภาพสินค้าดูโดดเด่นขึ้น
* **Accent Color:** ใช้สีแดง (`#e24242`) เป็นสีหลักสำหรับ Call-to-Action (ปุ่มสั่งซื้อ, ไอคอนแจ้งเตือน, ตัวอักษรเน้นย้ำ) ช่วยดึงดูดสายตาและสร้างความรู้สึกโฉบเฉี่ยวเข้ากับธีมสินค้ายานยนต์

### 2. การจัดวางตัวอักษร (Typography)
* ใช้ฟอนต์ **Prompt** (Google Fonts) ทั้งระบบ ซึ่งเป็นฟอนต์ไม่มีหัว (Sans-serif) ที่ให้ความรู้สึกทันสมัย อ่านง่ายทั้งบนจอขนาดเล็กและใหญ่
* ใช้ฟอนต์ **Novusa** สไตล์สปอร์ต สำหรับโลโก้ `CG RACING` โดยเฉพาะ เพื่อสร้าง Brand Identity ที่แข็งแกร่ง

### 3. การออกแบบที่รองรับทุกหน้าจอ (Responsive & App-like Experience)
* **Desktop & Tablet:** ใช้ Sidebar ด้านซ้ายสำหรับการนำทาง (Navigation) ที่สามารถขยายออกได้เมื่อโฮเวอร์ (Hover) ช่วยประหยัดพื้นที่แสดงผล
* **Mobile Experience:** เมื่อเปิดบนมือถือ (ความกว้างน้อยกว่า 768px) Sidebar จะถูกซ่อน และเปลี่ยนเป็น **Bottom Navigation Bar** แทน ให้ความรู้สึกเหมือนกำลังใช้งาน Mobile Application ทำให้ผู้ใช้กดเมนูได้ง่ายด้วยมือเดียว

### 4. สถาปัตยกรรม UI และเอฟเฟกต์ภาพ (Visual Effects & Glassmorphism)
* **Glassmorphism:** นำเทคนิค `backdrop-filter: blur()` มาใช้กับแถบ Navbar และ Sidebar ทำให้พื้นหลังเบลอและโปร่งแสงเล็กน้อย ดูมีมิติและหรูหรา
* **Cards & Shadows:** สินค้าแต่ละชิ้นถูกจัดอยู่ในรูปแบบ Card (`.product`) แยกสัดส่วนชัดเจน พร้อมใช้เงาแบบนุ่มนวล (`box-shadow`) สร้างความลึกให้กับหน้าเว็บ

### 5. ปฏิสัมพันธ์และการตอบสนอง (Micro-interactions & Animations)
* **Hover Effects:** ปุ่มและการ์ดสินค้ามีการตอบสนองเมื่อผู้ใช้ชี้เมาส์ (เลื่อนขึ้นเล็กน้อย, เปลี่ยนสีขอบ) ทำให้ผู้ใช้รู้ว่าองค์ประกอบนั้นสามารถคลิกได้
* **Hero Banner Animation:** รูปภาพบนแบนเนอร์หน้าหลักมีการใช้ `@keyframes floatProduct` ทำให้ภาพสินค้าลอยขึ้น-ลงอย่างนุ่มนวล ดึงดูดความสนใจตั้งแต่แรกเข้าเว็บ
* **Smart Feedback:** ใช้ไลบรารี `SweetAlert2` แทนที่ Alert มาตรฐานของเบราว์เซอร์ เพื่อให้ Pop-up แจ้งเตือน (เช่น ยืนยันการสั่งซื้อ, ลบสินค้า) มีความสวยงาม กลมกลืนกับธีม และเป็นมิตรกับผู้ใช้งานมากขึ้น

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)
* **Frontend:** HTML5, CSS3 (CSS Variables สำหรับ Theming), Vanilla JavaScript (ES6+)
* **Database (Mock):** JSON format เก็บในตัวแปร และ Browser LocalStorage
* **Icons:** FontAwesome
* **UI Components:** SweetAlert2 สำหรับหน้าต่างแจ้งเตือนและ Dialog

## 📂 โครงสร้างไฟล์ (File Structure)
* `index.html` - หน้าหลักของเว็บไซต์ (สไลเดอร์, สินค้าแนะนำ)
* `discover.html` - หน้าค้นหาและกรองสินค้าทั้งหมด
* `view_product.html` - หน้ารายละเอียดสินค้าแต่ละชิ้น
* `product_cart.html` - หน้าตะกร้าสินค้าและสรุปยอด
* `seller.html` - หน้าจัดการสินค้าสำหรับผู้ขาย (Dashboard)
* `login.html` / `register.html` - หน้าเข้าสู่ระบบและสมัครสมาชิก
* `contact.html` - หน้าข้อมูลการติดต่อร้านค้า
* `nav.html` - โค้ดส่วนแถบนำทาง (ใช้วิธี Fetch ดึงไปแสดงผล)
* `style.css` - ไฟล์จัดการความสวยงาม (Styling) และ CSS Variables
* `auth.js` - สคริปต์ตรวจสอบและจัดการการเข้าสู่ระบบ
* `products.js` - แหล่งข้อมูลสินค้าจำลอง (Array of Objects)
